A critical remote code execution (RCE) vulnerability, tracked as **CVE-2025-55182** (also known as React2Shell), was disclosed on December 3, 2025, in React Server Components (RSC). This flaw has a maximum severity rating of **CVSS 10.0**.

**Source**: React Security Advisory, Unit 42 Threat Brief

### **Core Vulnerability**
The vulnerability is an **unauthenticated RCE** caused by **insecure deserialization** within the React Flight protocol. An attacker can send a single, specially crafted HTTP request to a server function endpoint, allowing them to execute arbitrary code with the privileges of the server process. Testing indicates the exploit has **near-100% reliability** and works against **default configurations**.

**Source**: Unit 42 Analysis, Technical Deep Dives (Wiz, LogRocket)

### **Affected Versions and Scale**
*   **React**: Versions 19.0.0, 19.1.0, 19.1.1, and 19.2.0 of the `react-server-dom-*` packages.
*   **Next.js**: Versions 15.x, 16.x (App Router), and Canary builds from 14.3.0-canary.77.
*   **Other Frameworks**: Any framework bundling vulnerable React packages, including React Router, Waku, RedwoodSDK, Parcel, and Vite RSC plugins.

The impact is massive due to React's widespread adoption:
*   Used by roughly **40% of all developers**.
*   Palo Alto Networks identified over **968,000 React and Next.js instances** exposed on the public internet.
*   Wiz Research data shows **39% of cloud environments** contain vulnerable instances, with **44% having publicly exposed Next.js instances**.

**Sources**: Palo Alto Networks Cortex Xpanse Telemetry, Wiz Research Data, Industry Adoption Statistics

### **Observed Exploitation and Post-Exploitation Activity**
Active exploitation began within **20 hours** of public disclosure. Unit 42 and other threat intelligence firms observed a wide range of post-exploitation activities, including:
*   **Reconnaissance**: Automated scanning and fingerprinting of compromised systems.
*   **Malware Deployment**: Installation of cryptominers (XMRig), backdoors (SNOWLIGHT, VShell, KSwapDoor, Noodle RAT), and web shells.
*   **Credential Theft**: Attempts to harvest cloud credentials (AWS, Azure, GCP) and API keys.
*   **Threat Actors**: Activity linked to suspected **Chinese state-sponsored groups** (e.g., CL-STA-1015/UNC5174) and **North Korean-linked campaigns** using tools like EtherRAT.

**Sources**: Google Threat Intelligence Group, Microsoft Defender Research, Unit 42 Updates

### **Critical Mitigation Steps**
**Immediate patching is the only definitive mitigation.**
*   **React**: Upgrade to **19.0.1, 19.1.2, or 19.2.1** (and later to 19.2.3 for subsequent DoS fixes).
*   **Next.js**: Upgrade to the latest patched stable versions (**16.0.7, 15.5.7**, etc., as per the vendor's advisory).

**Sources**: React Blog Patch Notes, Next.js Security Advisory

Its exploitation leads directly to full server compromise, with state-sponsored and cybercriminal groups actively deploying diverse malware payloads. Patching affected React and Next.js applications is an urgent priority.
